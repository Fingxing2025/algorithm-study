# 倍增法复习文档

题单来源：P3865 / P5465 / P3295 / P2251 / P1816 / P1198 / P2880 / P5012。

这组题非常适合把“倍增法”拆成五个层次去复习：

1. 静态区间查询中的 ST 表。
2. 在线过程中的“尾插型倍增”。
3. 区间约束上的“长度按 $2^k$ 分层”。
4. 图或区间扩展中的“跳跃状态 + 累积贡献”。
5. 离线预处理后的答案轴查询与块上 ST 表。

国赛里，倍增法不等于“只会 LCA”。真正重要的是：

- 你是否能把问题抽成“长度 / 步数 / 区间规模”每次翻倍。
- 你是否知道什么时候可以直接合并两个 $2^{k-1}$ 状态。
- 你是否能判断查询时该用“重叠两块”还是“二进制拆块”。

---

## 1. 倍增法的本质

倍增的通用状态长这样：

- `f[x][k]`：从状态 `x` 出发，经过 `2^k` 长度、`2^k` 步、或覆盖 `2^k` 规模后得到的信息。

其核心转移几乎总是：

```text
f[x][k] <- merge(f[x][k-1], f[next(x, 2^(k-1))][k-1])
```

或者在静态区间问题中：

```text
st[i][k] <- op(st[i][k-1], st[i + 2^(k-1)][k-1])
```

你要先判断两件事：

1. 信息能否由两个半段合成。
2. 合成后的信息是否仍然足够支持后续查询。

## 3. ST 表专题

### 3.1 什么场景适合 ST 表

满足这三个条件时，优先想到 ST 表：

- 数组静态，不修改。
- 查询很多。
- 区间信息可快速合并。

最常见是：

- `max`
- `min`
- `gcd`

注意：

- 如果运算满足可重复贡献，查询可直接用两块重叠区间。
- 如果运算不允许重复贡献，就不能套 RMQ 那套两块重叠写法。

### 3.2 ST 表预处理

定义：`st[i][k]` 表示区间 `[i, i + 2^k - 1]` 的答案。

转移：

```cpp
for (int k = 1; k <= LG; ++k) {
    for (int i = 1; i + (1 << k) - 1 <= n; ++i) {
        st[i][k] = max(st[i][k - 1], st[i + (1 << (k - 1))][k - 1]);
    }
}
```

查询 `[l, r]`：

令 `k = floor(log2(r - l + 1))`，则

```cpp
ans = max(st[l][k], st[r - (1 << k) + 1][k]);
```

原因：长度为 `2^k` 的两段可以覆盖整个区间，且 `max/min/gcd` 重复合并不出错。

### 3.3 log 表

比赛里不要每次现算对数，直接预处理：

```cpp
lg[1] = 0;
for (int i = 2; i <= n; ++i) lg[i] = lg[i >> 1] + 1;
```

### 3.4 易错点

- `st` 的第二维开到 `lg[n]` 即可，但数组一般直接开 `20` 或 `25`。
- 注意下标从 `1` 开始时的右端点范围。
- `query(l, r)` 里不要把右侧块写成 `st[r][k]`。
- 时间卡得紧时，`O(n log n + q)` 的 ST 通常没问题，但常数要稳。

## 4. 三道基础 ST 题怎么串起来

### 4.1 P3865：模板题，练标准写法

题意：静态区间最大值。

复习重点：

- 记熟 `st[i][k]` 的定义。
- 记熟两块重叠查询。
- 记熟 `lg` 预处理。

一句话总结：

> P3865 是“把 ST 表写对”的题，不是“把 ST 表想出来”的题。

### 4.2 P1816：把 `max` 换成 `min`

题意：多次询问区间最小值。

它本质与 P3865 完全一致，只是运算从 `max` 变成 `min`。

复习价值：

- 让你确认自己写的是“可复用模板”，而不是一份只会求最大值的代码。

### 4.3 P2880：双信息同步维护

题意：询问区间内最大值与最小值之差。

做法：

- 一张 `stMax`。
- 一张 `stMin`。
- 查询时做一次最大值查询和一次最小值查询，答案相减。

复习重点：

- 同一个 RMQ 问题里，同时维护两个幂级层信息。
- 这类题说明倍增法本身并不限制你只维护一种值。

## 5. P2251：这题能用倍增，但你要知道更优做法

题意：求所有长度为 `m` 的滑动窗口最小值。

### 5.1 ST 做法

因为数组静态，窗口查询很多，所以完全可以：

- 先建 `min` 的 ST 表。
- 对每个窗口 `[i, i + m - 1]` 做一次 RMQ。

复杂度：

- 预处理 `O(n log n)`
- 查询总计 `O(n)`

### 5.2 更优正解

这题更常规、更优雅的正解是单调队列，复杂度 `O(n)`。

### 5.3 这题的复习价值

这题的价值不是“用 ST 过题”，而是建立判断力：

- 静态区间最值，ST 能做。
- 固定窗口最值，单调队列往往更优。

国赛里，很多人会“看见区间就 ST”，这会丢掉最优复杂度和更清晰的结构。

## 6. P1198：在线尾插型倍增

题意：支持两种操作：

- 在末尾插入一个数。
- 查询当前序列最后 `L` 个数的最大值。

这题常见正解是线段树，但它其实也很适合拿来训练“在线倍增”。

### 6.1 状态设计

定义：`mx[i][k]` 表示“以 `i` 为右端点，长度为 `2^k` 的后缀块最大值”。

也就是区间：

```text
[i - 2^k + 1, i]
```

### 6.2 在线插入转移

插入第 `i` 个数后：

```cpp
mx[i][0] = a[i];
for (int k = 1; (1 << k) <= i; ++k) {
    mx[i][k] = max(mx[i][k - 1], mx[i - (1 << (k - 1))][k - 1]);
}
```

这就是“从两个长度为 `2^{k-1}` 的后缀块拼出长度 `2^k` 的后缀块”。

### 6.3 查询最后 `L` 个数

因为查询区间总是后缀，所以可以把 `L` 做二进制拆分：

```cpp
int query(int right, int len) {
    int ans = 0;
    int cur = right;
    for (int k = LG; k >= 0; --k) {
        if ((1 << k) <= len) {
            ans = max(ans, mx[cur][k]);
            cur -= 1 << k;
            len -= 1 << k;
        }
    }
    return ans;
}
```

### 6.4 这题训练什么

- 不是所有倍增都建立在静态数组上。
- 只要“新增状态只依赖更短块”，就能在线维护。
- 查询如果不能直接用重叠两块，也可以走二进制拆块。

这点非常关键：

> ST 查询是“拿两块覆盖”。
> 在线后缀倍增查询是“把区间拆成若干块”。

两者都是倍增，但查询方式不同。

## 7. P3295：区间相等约束的分层倍增

题意：给若干个约束，要求两个等长区间对应位置完全相同，问满足条件的数字串个数。

这题是非常典型的“高级倍增题”。

### 7.1 关键观察

如果两个长度为 `len` 的区间相等，不必逐位连边。

只要取：

- `k = floor(log2(len))`
- 两个区间的前 `2^k` 段相等
- 两个区间的后 `2^k` 段相等

就能覆盖整个区间。

这就是把“大区间相等”压缩成“若干个 $2^k$ 长度区间相等”。

### 7.2 分层建点思路

把每个区间节点表示成：

- `node(i, k)`：起点为 `i`、长度为 `2^k` 的区间。

然后做分层并查集：

- 同一层上，维护哪些 `2^k` 区间被判定相等。
- 从大层向小层下传。

### 7.3 两种合并

#### 第一种：由题目约束触发的合并

若 `[l1, r1]` 与 `[l2, r2]` 相等，长度为 `len`。

取 `k = floor(log2(len))`，则合并：

- `node(l1, k)` 与 `node(l2, k)`
- `node(r1 - 2^k + 1, k)` 与 `node(r2 - 2^k + 1, k)`

#### 第二种：由“大块相等”向下分裂

如果两个长度 `2^k` 的区间相等，那么它们的：

- 左半段长度 `2^{k-1}` 相等
- 右半段长度 `2^{k-1}` 相等

所以从大层往下推时，再合并：

- `node(i, k-1)` 与 `node(j, k-1)`
- `node(i + 2^{k-1}, k-1)` 与 `node(j + 2^{k-1}, k-1)`

### 7.4 最后怎么计数

当所有关系都下传到 `k = 0` 时：

- `node(i, 0)` 就是第 `i` 位。
- 同一并查集连通块内的位必须填相同数字。

设最终有 `cnt` 个连通块：

- 最高位所在块不能填 `0`，有 `9` 种。
- 其余每个块有 `10` 种。

答案是：

```text
9 * 10^(cnt - 1)
```

### 7.5 这题最值得记住的点

> 倍增不一定是“跳父亲”。
> 也可以是“把长度为 `len` 的区间关系，压到若干层 `2^k` 区间节点上”。

这是国赛很常见的抽象方式。

## 8. P5465：区间图上的状态倍增

这题比前面所有题都更接近国赛难度本体。

如果你复习倍增法时只停留在 ST 或 LCA，这题基本无从下手。它真正训练的是：

- 你能不能自己定义“什么叫一步状态”。
- 你能不能把“一步转移”扩成“`2^k` 步转移”。
- 你能不能在跳跃表里同时维护边界与贡献。

### 8.1 复习时要抓住的抽象

这类题通常不是维护“某个点的祖先”，而是维护：

- 从某个起点或某个状态出发，经过若干次扩展后，能覆盖到的区间边界。
- 以及在这 `2^k` 次扩展过程中，累计产生了多少距离贡献、多少答案贡献。

也就是说，倍增表可能长成这样：

- `go[x][k]`：从状态 `x` 连续做 `2^k` 次扩展后到达的新状态。
- `sum[x][k]`：这 `2^k` 次扩展带来的累计贡献。

### 8.2 这题的复习价值

这题建议你重点复盘三件事：

1. 一步转移后，状态是否仍保持某种可压缩结构，比如连续区间。
2. 能否先预处理“一步转移”，再做倍增。
3. 查询时能否像 LCA 一样按位贪心，把答案拆成若干个 `2^k` 跳跃的和。

### 8.3 你应该从这题学到什么

不是去背细节，而是形成下面这个反应：

> 当题目存在“重复扩展”“重复跳跃”“重复覆盖”时，先问自己：
> 我能不能把一步的信息做成状态，再把状态倍增。

这就是国赛倍增题和普及组 ST 模板题的本质差别。

## 9. 两种查询方式必须分清

### 9.1 重叠两块

适用于：

- `max`
- `min`
- `gcd`

这种运算允许重复统计，所以长度 `len` 的区间可以直接拿两块长度 `2^k` 的区间覆盖。

### 9.2 二进制拆块

适用于：

- 查询过程必须按顺序拼接。
- 或者你的状态定义本来就是“后缀块 / 跳跃块 / 路径块”。

典型就是 P1198。

很多人学倍增时只记住 ST 的查询写法，结果遇到在线题就不会转了。

### 10.1 静态 ST 表模板

### 10.2 在线尾插倍增模板

会写：

- 新元素加入后的逐层更新

### 10.3 区间分层倍增模板

会写：

- 把区间长度离散到 `2^k`
- 大层约束向小层下传
- 与并查集、图、DP 结合

### 10.4 离线答案轴 RMQ 模板

会写：

- 先把原问题离线化成 `best[k]` 形式的答案轴。
- 再对答案轴做分块，维护块最优。
- 最后只在块数组上建 ST 表。

## 12. 常见误区

### 误区 1：倍增法 = LCA

错。LCA 只是“按步数翻倍跳父亲”的一个最经典版本。

### 误区 2：ST 表 = 所有区间问题最优解

错。P2251 就是反例。

### 误区 3：查询一定是两块重叠

错。P1198 这种在线后缀查询要二进制拆块。

### 误区 4：倍增状态只能是单点

错。P3295 的状态是区间块，P5465 的状态本质上更接近“扩展后的覆盖状态”。

---

## 13. 最后一页速记

如果你在考场上看到下面这些信号，要立刻想到倍增：

- “静态，多次区间最值” -> ST 表。
- “只增不删，查询后缀 / 前缀” -> 在线倍增。
- “区间长度相同、关系传递” -> 按 `2^k` 分层建点。
- “一步扩展能定义清楚，多次扩展要快速做” -> 状态倍增。
- “询问限制的是某个统计量范围” -> 先离线压成答案轴，再做 RMQ / 块上 ST。

一句总纲：

> 倍增法的重点不是 `2^k` 这个形式本身，而是把问题抽象成“半段能拼整段，少步能拼多步”。

如果这句话你在做 P1198、P3295、P5465 时都能真正用出来，倍增这部分才算复习到位。

## 14. 逐题完整代码与分析

这一章直接给出可交的 C++17 代码，并强调每题里倍增到底落在什么状态上。

### 14.1 P3865 【模板】ST 表 & RMQ 问题

核心分析：

- 状态是 `st[i][k]`，表示区间 `[i, i + 2^k - 1]` 的最大值。
- 查询用两块长度相同的重叠区间覆盖整个 `[l, r]`。
- 这是最标准的静态倍增。

完整代码：

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 100000 + 5;
const int LOG = 20;

int n, m;
int st[MAXN][LOG], lg2_[MAXN];

int query(int l, int r) {
    int k = lg2_[r - l + 1];
    return max(st[l][k], st[r - (1 << k) + 1][k]);
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n >> m;
    for (int i = 1; i <= n; ++i) cin >> st[i][0];

    lg2_[1] = 0;
    for (int i = 2; i <= n; ++i) lg2_[i] = lg2_[i >> 1] + 1;

    for (int k = 1; k < LOG; ++k) {
        for (int i = 1; i + (1 << k) - 1 <= n; ++i) {
            st[i][k] = max(st[i][k - 1], st[i + (1 << (k - 1))][k - 1]);
        }
    }

    while (m--) {
        int l, r;
        cin >> l >> r;
        cout << query(l, r) << '\n';
    }
    return 0;
}
```

代码分析：

- 预处理复杂度 `O(n log n)`。
- 单次查询复杂度 `O(1)`。
- 这题的目标不是创新，而是把 ST 模板写到不出边界错。

### 14.2 P1816 忠诚

核心分析：

- 与 P3865 完全同构，只是把 `max` 替换成 `min`。
- 这题适合检查你的 ST 模板是否足够抽象。

完整代码：

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 100000 + 5;
const int LOG = 20;

int n, m;
int st[MAXN][LOG], lg2_[MAXN];

int query(int l, int r) {
    int k = lg2_[r - l + 1];
    return min(st[l][k], st[r - (1 << k) + 1][k]);
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n >> m;
    for (int i = 1; i <= n; ++i) cin >> st[i][0];

    lg2_[1] = 0;
    for (int i = 2; i <= n; ++i) lg2_[i] = lg2_[i >> 1] + 1;

    for (int k = 1; k < LOG; ++k) {
        for (int i = 1; i + (1 << k) - 1 <= n; ++i) {
            st[i][k] = min(st[i][k - 1], st[i + (1 << (k - 1))][k - 1]);
        }
    }

    for (int i = 1; i <= m; ++i) {
        int l, r;
        cin >> l >> r;
        cout << query(l, r) << (i == m ? '\n' : ' ');
    }
    return 0;
}
```

代码分析：

- 唯一变化是聚合函数。
- 国赛里常见的是把 `max/min/gcd` 都塞进同一套 RMQ 思维里。

### 14.3 P2880 Balanced Lineup G

核心分析：

- 本质还是 RMQ。
- 倍增状态变成两张表同时维护：`stMax` 和 `stMin`。
- 查询时分别求区间最大最小，再做差。

完整代码：

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 50000 + 5;
const int LOG = 17;

int n, q;
int stMax[MAXN][LOG + 1], stMin[MAXN][LOG + 1], lg2_[MAXN];

int queryMax(int l, int r) {
    int k = lg2_[r - l + 1];
    return max(stMax[l][k], stMax[r - (1 << k) + 1][k]);
}

int queryMin(int l, int r) {
    int k = lg2_[r - l + 1];
    return min(stMin[l][k], stMin[r - (1 << k) + 1][k]);
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n >> q;
    for (int i = 1; i <= n; ++i) {
        cin >> stMax[i][0];
        stMin[i][0] = stMax[i][0];
    }

    lg2_[1] = 0;
    for (int i = 2; i <= n; ++i) lg2_[i] = lg2_[i >> 1] + 1;

    for (int k = 1; k <= LOG; ++k) {
        for (int i = 1; i + (1 << k) - 1 <= n; ++i) {
            stMax[i][k] = max(stMax[i][k - 1], stMax[i + (1 << (k - 1))][k - 1]);
            stMin[i][k] = min(stMin[i][k - 1], stMin[i + (1 << (k - 1))][k - 1]);
        }
    }

    while (q--) {
        int l, r;
        cin >> l >> r;
        cout << queryMax(l, r) - queryMin(l, r) << '\n';
    }
    return 0;
}
```

代码分析：

- 这题是“双信息倍增”的基础题。
- 只要每种信息都满足可合并，就可以并行建多张 ST 表。

### 14.4 P2251 质量检测

核心分析：

- 从倍增角度，这题可以直接建最小值 ST 表。
- 然后枚举每个窗口 `[i, i + m - 1]` 做一次 RMQ。
- 但你必须知道，最优正解其实是单调队列 `O(n)`。

这里给出倍增版代码，因为这份文档的主题是倍增法。

完整代码：

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 100000 + 5;
const int LOG = 20;

int n, m;
int st[MAXN][LOG], lg2_[MAXN];

int query(int l, int r) {
    int k = lg2_[r - l + 1];
    return min(st[l][k], st[r - (1 << k) + 1][k]);
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n >> m;
    for (int i = 1; i <= n; ++i) cin >> st[i][0];

    lg2_[1] = 0;
    for (int i = 2; i <= n; ++i) lg2_[i] = lg2_[i >> 1] + 1;

    for (int k = 1; k < LOG; ++k) {
        for (int i = 1; i + (1 << k) - 1 <= n; ++i) {
            st[i][k] = min(st[i][k - 1], st[i + (1 << (k - 1))][k - 1]);
        }
    }

    for (int i = 1; i + m - 1 <= n; ++i) {
        cout << query(i, i + m - 1) << '\n';
    }
    return 0;
}
```

代码分析：

- 这份代码的复杂度是 `O(n log n)` 预处理加 `O(n)` 查询。
- 做题时如果你第一反应是 ST，说明倍增意识到了。
- 但国赛层面还要继续追问一句：有没有更优的线性算法。

### 14.5 P1198 最大数

核心分析：

- 这题的倍增不是静态区间，而是“在线后缀块”。
- `mx[i][k]` 表示以 `i` 为右端点、长度为 `2^k` 的块最大值。
- 插入新元素时只更新第 `i` 行；查询最后 `L` 个数时做二进制拆块。

完整代码：

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXM = 200000 + 5;
const int LOG = 20;

int m;
long long modValue;
long long mx[MAXM][LOG];
int lengthNow = 0;
long long lastAnswer = 0;

long long querySuffixMax(int len) {
    long long ans = 0;
    int cur = lengthNow;
    for (int k = LOG - 1; k >= 0; --k) {
        if ((1 << k) <= len) {
            ans = max(ans, mx[cur][k]);
            cur -= 1 << k;
            len -= 1 << k;
        }
    }
    return ans;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> m >> modValue;
    while (m--) {
        char op;
        long long x;
        cin >> op >> x;
        if (op == 'A') {
            long long value = (x + lastAnswer) % modValue;
            if (value < 0) value += modValue;
            ++lengthNow;
            mx[lengthNow][0] = value;
            for (int k = 1; k < LOG && (1 << k) <= lengthNow; ++k) {
                mx[lengthNow][k] = max(mx[lengthNow][k - 1], mx[lengthNow - (1 << (k - 1))][k - 1]);
            }
        } else {
            lastAnswer = querySuffixMax((int)x);
            cout << lastAnswer << '\n';
        }
    }
    return 0;
}
```

代码分析：

- 这题要背的是“新增一个点时，如何反向建出以它为右端点的所有幂长块”。
- 查询不是 ST 的“重叠两块”，而是按二进制把后缀拆成若干块。
- 这正是倍增题里最容易和 ST 混掉的地方。

### 14.6 P3295 萌萌哒

核心分析：

- 状态不是单点，而是 `node(pos, k)`，表示从 `pos` 开始长度为 `2^k` 的区间。
- 同一层用并查集维护哪些区间块相等。
- 约束先合并大块，再从高层往低层把等价关系下传。

完整代码：

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 100000 + 5;
const int LOG = 18;
const int MOD = 1000000007;

int n, m;
int parent_[LOG][MAXN];

int findSet(int level, int x) {
    if (parent_[level][x] == x) return x;
    return parent_[level][x] = findSet(level, parent_[level][x]);
}

void unite(int level, int x, int y) {
    x = findSet(level, x);
    y = findSet(level, y);
    if (x != y) parent_[level][x] = y;
}

long long modPow(long long a, long long b) {
    long long res = 1;
    while (b > 0) {
        if (b & 1) res = res * a % MOD;
        a = a * a % MOD;
        b >>= 1;
    }
    return res;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n >> m;
    for (int k = 0; k < LOG; ++k) {
        for (int i = 1; i <= n; ++i) {
            parent_[k][i] = i;
        }
    }

    while (m--) {
        int l1, r1, l2, r2;
        cin >> l1 >> r1 >> l2 >> r2;
        int len = r1 - l1 + 1;
        for (int k = LOG - 1; k >= 0; --k) {
            if ((1 << k) <= len) {
                unite(k, l1, l2);
                l1 += 1 << k;
                l2 += 1 << k;
                len -= 1 << k;
            }
        }
    }

    for (int k = LOG - 1; k >= 1; --k) {
        int half = 1 << (k - 1);
        int limit = n - (1 << k) + 1;
        for (int start = 1; start <= limit; ++start) {
            int root = findSet(k, start);
            if (root == start) continue;
            unite(k - 1, start, root);
            unite(k - 1, start + half, root + half);
        }
    }

    int components = 0;
    for (int i = 1; i <= n; ++i) {
        if (findSet(0, i) == i) ++components;
    }

    long long answer = 9LL * modPow(10, components - 1) % MOD;
    cout << answer << '\n';
    return 0;
}
```

代码分析：

- 读入约束时，采用二进制拆长度的方式，把任意等长区间拆成若干个 `2^k` 块。
- 下传阶段本质像线段树的 `pushdown`，但这里推的是“相等关系”。
- 统计答案时只看 `k = 0` 层，因为它已经对应到每一位数字。

你要重点理解这句：

> 大区间相等，不是逐位合并，而是先在高层做块级合并，再层层拆到底层。

这就是区间倍增最重要的抽象。

### 14.7 P5465 星际穿越

核心分析：

- 这题的关键不是“某个点跳 `2^k` 次到哪”，而是“从区间 `[i, n]` 出发，经过 `2^k` 次扩展后，最左能覆盖到哪，以及这段过程中产生了多少距离和”。
- 设 `jumpTo[i][k]` 为从状态 `[i, n]` 出发做 `2^k` 次扩展后能到达的最左端点。
- 设 `costSum[i][k]` 为这 `2^k` 次扩展，对区间 `(jumpTo[i][k], i)` 内所有点贡献的最短路总和。

最容易错的点：

- `jumpTo[i][k]` 的状态起点是整个区间 `[i, n]`，不是单独的点 `i`。
- 查询时先强制让 `x` 走到 `l[x]`，再把剩余部分视作区间状态去倍增。

完整代码：

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 300000 + 5;
const int LOG = 19;

int n, q;
int leftReach[MAXN], suffixMinLeft[MAXN];
int jumpTo[MAXN][LOG];
long long costSum[MAXN][LOG];

long long gcdll(long long a, long long b) {
    while (b != 0) {
        long long t = a % b;
        a = b;
        b = t;
    }
    return a;
}

long long prefixDistanceSum(int leftBound, int x) {
    if (leftReach[x] <= leftBound) {
        return x - leftBound;
    }

    int usedSteps = 1;
    int cur = leftReach[x];
    long long res = x - leftReach[x];

    for (int k = LOG - 1; k >= 0; --k) {
        if (jumpTo[cur][k] >= leftBound) {
            res += costSum[cur][k] + 1LL * (cur - jumpTo[cur][k]) * usedSteps;
            usedSteps += 1 << k;
            cur = jumpTo[cur][k];
        }
    }

    res += 1LL * (cur - leftBound) * (usedSteps + 1);
    return res;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n;
    leftReach[1] = 1;
    for (int i = 2; i <= n; ++i) cin >> leftReach[i];

    suffixMinLeft[n + 1] = INT_MAX;
    for (int i = n; i >= 1; --i) {
        suffixMinLeft[i] = min(suffixMinLeft[i + 1], leftReach[i]);
    }

    for (int i = 1; i <= n; ++i) {
        jumpTo[i][0] = suffixMinLeft[i];
        costSum[i][0] = i - jumpTo[i][0];
    }

    for (int k = 1; k < LOG; ++k) {
        for (int i = 1; i <= n; ++i) {
            int mid = jumpTo[i][k - 1];
            jumpTo[i][k] = jumpTo[mid][k - 1];
            costSum[i][k] = costSum[i][k - 1]
                          + costSum[mid][k - 1]
                          + 1LL * (1 << (k - 1)) * (mid - jumpTo[i][k]);
        }
    }

    cin >> q;
    while (q--) {
        int l, r, x;
        cin >> l >> r >> x;
        long long numerator = prefixDistanceSum(l, x) - prefixDistanceSum(r + 1, x);
        long long denominator = r - l + 1;
        long long g = gcdll(numerator, denominator);
        cout << numerator / g << '/' << denominator / g << '\n';
    }
    return 0;
}
```

代码分析：

- `suffixMinLeft[i]` 是“一步扩展后的最左端点”，它把向右跳和向左跳统一进了一个区间状态。
- `jumpTo[i][k]` 维护的是做完 `2^k` 次扩展后的最左边界。
- `costSum[i][k]` 维护的是这一段扩展给距离和带来的贡献。
- 查询函数 `prefixDistanceSum(leftBound, x)` 的含义是求 `sum(dist(x, t))`，其中 `t` 从 `leftBound` 到 `x - 1`。
- 真正答案用差分：

```text
sum_{t=l}^{r} dist(x, t)
= prefixDistanceSum(l, x) - prefixDistanceSum(r + 1, x)
```

这题是整套题里最像国赛思维的：

- 先把一步操作改写成可压缩状态。
- 再把状态做倍增。
- 最后用按位贪心还原查询。

### 14.8 P5012 水の数列

核心分析：

- 这题看上去是“选一个阈值 `x`，再回答在线询问”，但真正的关键是先把题目离线化。
- 若固定阈值 `x`，把所有 `<= x` 的位置染黑，那么黑色连续段长度平方和记为 `raw`，真实目标是最大化 `raw / x`。
- 对于固定的连续段个数 `cnt`，只需要保留一个最优二元组 `(raw, x)`。
- 于是原问题被压成一条“答案轴”：`best[cnt]`，表示段数恰为 `cnt` 时的最优解。
- 询问给出的 `[l, r]`，本质上就是在答案轴区间 `[l, r]` 上做区间最优查询。

### 14.8.1 简化过程

第一步简化：`x` 只需要取数组里出现过的值。

- 因为当阈值落在两个相邻不同值之间时，染黑集合不变，但分母更大，答案不会更优。

第二步简化：按值从小到大扫描阈值。

- 每加入一个新值，就把该值出现的位置激活。
- 激活位置在下标轴上形成的连通块，就是当前黑色连续段。
- 用并查集维护：
  - 当前连续段个数 `cnt`
  - 当前平方和 `sum`

第三步简化：对每个 `cnt` 只保留最优 `(sum, x)`。

- 比较两个候选时，不比较 `sum` 本身，而比较 `sum / x`。
- 用交叉相乘避免浮点误差。

这样原题就被化成：

```text
best[1], best[2], ..., best[n]
```

每次询问只是在 `best[l..r]` 中找最优答案。

### 14.8.2 倍增到底用在什么地方

这题最值得复习的点是：

> 倍增不一定用在原始对象上，也可以用在“离线预处理之后得到的答案数组”上。

这里倍增没有直接用于原数组 `a[i]`，而是用于块数组：

- 先把答案轴 `best[1..n]` 分块。
- 每块维护一个块内最优 `mx[block]`。
- 再只对块数组 `mx` 建 ST 表。

这样做的原因是：

- 直接对 `best[1..n]` 建完整 ST 表，空间是 `O(n log n)`，常数偏大。
- 分块后块数约为 `n / 1024`，块上 ST 表的空间和时间都更稳。

查询时拆成三段：

- 左残块暴力扫。
- 右残块暴力扫。
- 中间整块用块上 ST 表查询。

这就是典型的“分块负责压空间，ST 表负责处理中间整块”的混合结构。

完整代码：

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;
const int N = 1000000 + 5;
const int V = 1000000 + 5;
const int S = 1024;
const int B = 2005;
const int LG = 12;

int n, q, a[N], head[V], nxt[N], fa[N], sz[N], c[N], bel[N], L[B], R[B], st[B][LG], Log2[B];
int maxa;
ll w[N], lastans;
unsigned char vis[N];

inline int read()
{
    int x = 0, f = 1;
    char ch = getchar();
    while (ch < '0' || ch > '9')
    {
        if (ch == '-')
            f = -1;
        ch = getchar();
    }
    while (ch >= '0' && ch <= '9')
    {
        x = x * 10 + ch - '0';
        ch = getchar();
    }
    return x * f;
}

inline bool cmp(int x, int y)
{
    if (!x || !c[x])
        return false;
    if (!y || !c[y])
        return true;
    __int128 t1 = (__int128)w[x] * c[y];
    __int128 t2 = (__int128)w[y] * c[x];
    if (t1 != t2)
        return t1 > t2;
    return c[x] > c[y];
}

inline int find(int x)
{
    return fa[x] == x ? x : fa[x] = find(fa[x]);
}

int main()
{
    for (int i = 2; i < B; ++i)
        Log2[i] = Log2[i >> 1] + 1;

    n = read();
    q = read();
    for (int i = 1; i <= n; ++i)
    {
        a[i] = read();
        if (a[i] > maxa)
            maxa = a[i];
        nxt[i] = head[a[i]];
        head[a[i]] = i;
        fa[i] = i;
        sz[i] = 1;
    }

    ll sum = 0;
    int cnt = 0;
    for (int num = 1; num <= maxa; ++num)
    {
        if (!head[num])
            continue;
        for (int p = head[num]; p; p = nxt[p])
        {
            vis[p] = 1;
            ++cnt;
            ++sum;
            if (p > 1 && vis[p - 1])
            {
                int x = find(p - 1), y = find(p);
                if (x != y)
                {
                    if (sz[x] < sz[y])
                        swap(x, y);
                    sum -= 1LL * sz[x] * sz[x];
                    sum -= 1LL * sz[y] * sz[y];
                    fa[y] = x;
                    sz[x] += sz[y];
                    sum += 1LL * sz[x] * sz[x];
                    --cnt;
                }
            }
            if (p < n && vis[p + 1])
            {
                int x = find(p + 1), y = find(p);
                if (x != y)
                {
                    if (sz[x] < sz[y])
                        swap(x, y);
                    sum -= 1LL * sz[x] * sz[x];
                    sum -= 1LL * sz[y] * sz[y];
                    fa[y] = x;
                    sz[x] += sz[y];
                    sum += 1LL * sz[x] * sz[x];
                    --cnt;
                }
            }
        }

        if (!c[cnt])
        {
            w[cnt] = sum;
            c[cnt] = num;
        }
        else
        {
            __int128 t1 = (__int128)sum * c[cnt];
            __int128 t2 = (__int128)w[cnt] * num;
            if (t1 > t2 || (t1 == t2 && num > c[cnt]))
            {
                w[cnt] = sum;
                c[cnt] = num;
            }
        }
    }

    int tot = (n + S - 1) / S;
    for (int i = 1; i <= tot; ++i)
    {
        L[i] = (i - 1) * S + 1;
        R[i] = min(n, i * S);
        for (int j = L[i]; j <= R[i]; ++j)
        {
            bel[j] = i;
            if (cmp(j, st[i][0]))
                st[i][0] = j;
        }
    }

    for (int j = 1; (1 << j) <= tot; ++j)
    {
        for (int i = 1; i + (1 << j) - 1 <= tot; ++i)
        {
            int x = st[i][j - 1], y = st[i + (1 << (j - 1))][j - 1];
            st[i][j] = cmp(x, y) ? x : y;
        }
    }

    for (int i = 1; i <= q; ++i)
    {
        ll aa = read(), bb = read(), x = read(), y = read();
        ll lst = lastans;
        ll l = (aa * lst + x - 1) % n + 1;
        ll r = (bb * lst + y - 1) % n + 1;
        if (l > r)
            swap(l, r);

        int ans = 0;
        if (bel[l] == bel[r])
        {
            for (int j = (int)l; j <= (int)r; ++j)
            {
                if (cmp(j, ans))
                    ans = j;
            }
        }
        else
        {
            for (int j = (int)l; j <= R[bel[l]]; ++j)
            {
                if (cmp(j, ans))
                    ans = j;
            }
            for (int j = L[bel[r]]; j <= (int)r; ++j)
            {
                if (cmp(j, ans))
                    ans = j;
            }
            if (bel[r] - bel[l] > 1)
            {
                int l1 = bel[l] + 1, r1 = bel[r] - 1;
                int k = Log2[r1 - l1 + 1];
                int x1 = st[l1][k], x2 = st[r1 - (1 << k) + 1][k];
                int m = cmp(x1, x2) ? x1 : x2;
                if (cmp(m, ans))
                    ans = m;
            }
        }

        if (!ans)
        {
            printf("-1 -1\n");
            printf("%lld %lld %lld\n", l, r, lst);
            lastans = 1 % n;
        }
        else
        {
            printf("%lld %d\n", w[ans], c[ans]);
            printf("%lld %lld %lld\n", l, r, lst);
            lastans = (ll)((__int128)w[ans] * c[ans] % n);
        }
    }

    return 0;
}
```

代码分析：

- 预处理阶段的核心不是倍增，而是“按值离线 + 并查集维护连通块”。
- 真正的倍增用在块上 ST 表：`st[i][j]` 表示从第 `i` 块开始连续 `2^j` 块的最优答案。
- 查询时，倍增只处理完整块区间，两端零头用分块暴力扫掉。
- 这题很适合提醒自己：当原问题的查询对象很复杂时，先想能不能离线压成一个静态数组，再在那个静态数组上做 ST 表。
