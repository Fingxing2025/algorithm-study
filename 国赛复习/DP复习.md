# DP 复习文档

内容来源：模板库/dp 目录与背包问题子目录。

这组模板非常适合把动态规划拆成六个层次去复习：

1. 线性序列上的经典状态转移。
2. 计数型 DP 中的状态压缩与末尾信息维护。
3. 字符串 DP 中的二维前缀状态。
4. 区间 / 合并类 DP 中的“由小段拼大段”。
5. 背包类 DP 中的“物品维 + 容量维”转移。
6. 优化型 DP 中的滚动数组、单调队列、二进制拆分。

国赛里，DP 不等于“会背几道模板题”。真正重要的是：

- 你能不能先写出状态定义，再决定循环顺序。
- 你能不能判断当前题是“求最值”“计数”“可行性”还是“构造”。
- 你能不能分清楚哪些优化只是降常数，哪些优化会改变复杂度等级。

如果目标已经从“基础题会做”进入“国赛 DP 体系化补齐”，请继续看：DP进阶补齐清单.md。

## 1. 动态规划的本质

DP 的核心不是表格，而是两个问题：

1. 当前状态该如何描述，才能既不丢信息，又不冗余。
2. 当前状态能否由更小的状态转移而来。

最常见的状态长这样：

- `dp[i]`：处理到第 `i` 个位置时的最优值 / 方案数。
- `dp[i][j]`：处理到第 `i` 个位置，且附带一个额外维度 `j` 时的答案。
- `dp[i][j][k]`：如果前两维仍不足以表达限制，就继续补维。

通用转移一般是：

```text
dp[当前状态] <- merge(若干个更小状态)
```

所以做题顺序建议固定成这四步：

1. 先写状态含义。
2. 再写初始条件。
3. 再写转移来源。
4. 最后决定循环顺序和是否能优化。

> “状态定义先于代码，循环顺序服务于状态定义，优化只是在不改变状态语义的前提下减少无效转移。”

### 2.1 常见题型判定表

比赛里如果一时想不出状态，可以先用下面这张表做第一轮分类：

| 题面特征                                 | 第一反应              | 常见状态写法                         | 这一类最容易错在哪           |
| ---------------------------------------- | --------------------- | ------------------------------------ | ---------------------------- |
| 求最长上升 / 最长合法子序列              | 线性最值 DP           | `dp[i]` 表示以 `i` 结尾的最优值      | 答案误写成 `dp[n]`           |
| 两个字符串互相转换 / 匹配                | 二维前缀 DP           | `dp[i][j]` 表示前缀对前缀的答案      | 边界初始化没写全             |
| 统计满足限制的字符串个数                 | 计数 DP               | `dp[i][state]`                       | 状态记少了，下一位无法判断   |
| 题目含“末尾连续多少个”“最近几位长什么样” | 末尾信息 DP           | `dp[i][tail_state]`                  | 末尾状态定义不完整           |
| 若干物品选或不选，受容量限制             | 背包 DP               | `dp[j]` 或 `dp[j][k]`                | 容量循环顺序写反             |
| 每组最多选一个 / 附件依赖主件            | 分组或依赖背包        | 按组或按主件推进                     | 同组重复选取                 |
| 转移来自一个区间中的最优前驱             | 区间最值优化 DP       | `dp[i] = best(window) + cost`        | 忘记用单调队列或数据结构优化 |
| 最后一次操作覆盖一段区间 / 一段前缀      | 最后一步划分 DP       | `dp[i][j]` 或 `dp[i]` + 枚举最后一段 | 不会把问题缩成前缀           |
| 小块能合并成大块                         | 合并型 DP / 倍增式 DP | `f[level][pos]` 或 `dp[len][l]`      | 没有记录足够的拼接信息       |

更实战的判断顺序是：

1. 先看答案是最值、计数还是可行性。
2. 再看状态是按位置推进，还是按容量推进，还是按区间推进。
3. 最后再判断是否需要优化。

## 3. 线性序列 DP

### 3.1 LIS：最基础的线性最值 DP

模板：LIS.cpp

定义：`dp[i]` 表示以 `a[i]` 结尾的最长上升子序列长度。

转移：

```text
dp[i] = max(dp[j] + 1), 其中 j < i 且 a[j] < a[i]
```

复习重点：

- “以谁结尾”是 LIS 最经典的状态设计。
- 外层枚举终点 `i`，内层枚举上一个位置 `j`。
- 答案不是 `dp[n]`，而是所有 `dp[i]` 的最大值。

复杂度：`O(n^2)`。

这类题最适合训练：

- 如何从“整个序列最优”拆成“以某个位置结尾的最优”。
- 如何识别“最后一步来自哪里”。

### 3.2 LIS 的优化：贪心 + 二分

模板：LIS贪心+二分.cpp

这里已经不是传统意义上的 DP 表，而是维护：

- `t[len - 1]`：长度为 `len` 的上升子序列，其最小可能结尾值。

更新方式：

- 如果当前数比所有结尾都大，就扩展长度。
- 否则用 `lower_bound` 找到第一个大于等于它的位置替换。

复习重点：

- 这是把原本 `O(n^2)` 的状态转移压缩成了“只维护每个长度最优结尾”。
- 要理解“最小结尾更优”而不是死记模板。
- 这类优化常见于“只关心长度，不关心具体方案”的题。

一句话总结：

> 朴素 LIS 是“枚举上一个位置”，优化 LIS 是“维护每种长度的最优结尾”。

## 4. 字符串与计数型 DP

### 4.1 编辑距离：前缀对前缀

模板：修改字符串.cpp

定义：`dp[i][j]` 表示把 `s1` 的前 `i` 个字符变成 `s2` 的前 `j` 个字符的最小编辑距离。

三种转移：

1. 删除：`dp[i - 1][j] + 1`
2. 插入：`dp[i][j - 1] + 1`
3. 替换 / 不替换：`dp[i - 1][j - 1] + cost`

其中：

- 若 `s1[i - 1] == s2[j - 1]`，则 `cost = 0`
- 否则 `cost = 1`

复习重点：

- 二维字符串 DP 最常见的状态就是“前缀对前缀”。
- `dp[i][0] = i`、`dp[0][j] = j` 这种边界必须先写清楚。
- 看起来是三种操作，本质上都是问“最后一步做了什么”。

### 4.2 不含 101 的字符串个数：末尾信息状态化

模板：不含101字符串个数.cpp

这题本质是：

- 当限制涉及“不能出现某种子串”时，必须维护足够的末尾信息。

模板里用四个数组分类状态，本质上是在区分：

- 当前串末尾两位属于哪种形态。

复习重点：

- 这类题的关键不是数组名字，而是“要记住多少位末尾信息，才能判断下一位是否合法”。
- 如果限制模式更复杂，通常可以升级成自动机 DP。

一句话理解：

> 只要题目限制和“最近几位长什么样”有关，就考虑把末尾形态变成状态。

### 4.3 长度为 n，含至少一次连续 m 个 1 的字符串个数

模板：长n含m个1的字符串个数.cpp

定义：

- `dp[i][j]` 表示长度为 `i`，且末尾连续 1 的个数为 `j` 的合法串数。

转移：

1. 末尾补 `0`：前一个状态的任何 `j` 都可以转过来，并且新末尾连续 1 长度变成 `0`
2. 末尾补 `1`：只能从上一层的 `j - 1` 转过来

模板里求的是补集：

- 先求“不含连续 m 个 1”的方案数
- 再用总方案数减掉它

复习重点：

- 计数 DP 很常见的技巧是“正难则反”。
- 状态只需要维护“当前末尾连续 1 长度”，不必记完整串。
- 这里用了滚动数组，因为第 `i` 层只依赖第 `i - 1` 层。

## 5. 合并型 / 区间型 DP

### 5.1 倍增法应用：由两个小块拼成一个大块

模板：倍增法应用.cpp

定义：`f[x][i]` 表示从位置 `i` 开始，能否拼出一个值为 `x` 的区间；若能，则记录这段结束后的下一个位置。

转移思想：

- 如果从 `i` 开始有一段值为 `x-1` 的区间，且它后面紧接着还有一段值为 `x-1` 的区间
- 那么这两段可以合并成一段值为 `x` 的区间

这类题的本质不是倍增，而是：

- 小段信息足够时，大段信息可以由两个相邻小段合并得到。

复习重点：

- 这种状态很像“区间 DP + 倍增合并”的混合模型。
- `f[x][i]` 记录的不是布尔值，而是“合并成功后走到哪里”，这让后续拼接更方便。

### 5.2 区间覆盖问题：最后一段思维

模板：区间覆盖问题.cpp

定义：

- `dp[i][j]` 表示前 `i` 个位置里，恰好覆盖 `j` 个点的最小代价。

转移时通常分两种：

1. 第 `i` 个点不被最后一段覆盖，直接继承 `dp[i - 1][j]`
2. 枚举最后一段连续覆盖了多少个点，再把问题缩到前面的前缀

这就是很典型的“最后一段 / 最后一次决策”模型。

复习重点：

- 区间 DP 不一定非要写成 `dp[l][r]`。
- 只要问题是“前缀 + 最后一次操作”，就可以用前缀型二维 DP。
- 这类题常见瓶颈在“最后一段”枚举过多，需要再做预处理或优化。

## 6. 背包专题

### 6.1 0/1 背包

模板：01背包.cpp

定义：`dp[j]` 表示容量不超过 `j` 时的最大价值。

转移：

```text
dp[j] = max(dp[j], dp[j - w[i]] + v[i])
```

为什么容量要倒序？

- 因为每件物品最多选一次。
- 倒序可以保证当前物品不会在同一轮被重复使用。

这是背包题里最基础也最重要的一条规律。

### 6.2 完全背包

模板：完全背包.cpp

和 0/1 背包唯一关键区别：

- 每件物品可以选无限次。

所以容量要正序枚举。

原因：

- 正序时，`dp[j - w[i]]` 可能已经包含当前物品，从而实现重复选取。

一句话记忆：

- 0/1 背包倒序。
- 完全背包正序。

### 6.3 多重背包

模板：多重背包.cpp

每种物品最多选 `num` 件。

朴素做法要再枚举选几件，通常太慢。

模板用了二进制拆分：

- 把 `num` 件拆成 `1, 2, 4, ..., 剩余量`
- 每一块都看成一件新的 0/1 物品

这样总物品数从 `num` 降到 `log num` 级别。

复习重点：

- 二进制拆分本质上是把“有限次重复”转成若干个“选或不选”。
- 拆完后，仍然做 0/1 背包，容量倒序。

### 6.4 分组背包

模板：分组背包.cpp

题意模型：

- 物品按组划分
- 每组最多选一件

转移方式：

- 外层枚举组
- 中层倒序枚举容量
- 内层枚举这一组选哪件

复习重点：

- 关键不是“按组存数据”，而是“这一轮转移只能基于上一组的结果”。
- 所以容量仍然倒序，防止同一组里选多件。

### 6.5 二维 0/1 背包

模板：二维01.cpp

定义：`dp[j][k]` 表示在容量约束 `j`、时间约束 `k` 下的最大价值。

转移时两个维度都要倒序：

- 否则同一件物品会被重复使用。

复习重点：

- 多维费用背包只是“多开几个约束维度”，本质仍是 0/1 背包。
- 每增加一个“每件物品只能选一次”的约束维度，就多一个倒序循环。

### 6.6 依赖背包 / 主件附件背包

模板：依赖背包.cpp

这里的依赖关系是：

- 附件不能单独买，必须跟主件一起买。

模板做法是把每个主件的合法购买方式枚举出来：

1. 只买主件
2. 主件 + 附件 1
3. 主件 + 附件 2
4. 主件 + 两个附件都买

然后把这四种合法组合当作转移选项。

复习重点：

- 依赖背包的难点往往不在背包，而在“先把合法组合列出来”。
- 一旦组合整理完成，本质还是一轮 0/1 背包。

## 7. 优化型 DP

### 7.1 滚动数组

在这份模板里，长n含m个1的字符串个数.cpp 已经用了滚动数组。

使用条件：

- 第 `i` 层状态只依赖第 `i - 1` 层，或者只依赖有限几层。

优点：

- 把空间从 `O(nm)` 压到 `O(m)` 或 `O(常数 * m)`。

注意：

- 若某一层的转移会反复读取当前层旧值，就不能随便滚。
- 滚动后要特别小心哪些格子需要清空。

### 7.2 单调队列优化线性 DP

模板：单调队列优化线性dp.cpp

定义：`dp[i]` 表示到达位置 `i` 时的最大得分。

若转移形如：

```text
dp[i] = a[i] + max(dp[j]), 其中 j 属于某个滑动窗口
```

就可以用单调队列维护窗口内的最优 `dp[j]`。

模板做了三件事：

1. 新元素进窗口时，维护队列单调性。
2. 队头过期时弹出。
3. 当前答案直接取队头。

复习重点：

- 单调队列优化的前提是“转移只和一个滑动窗口内的最值有关”。
- 队列里存的是下标，不是值。
- 要同时维护“值的单调性”和“下标是否过期”。

一句话总结：

> 当 DP 转移从“枚举一段区间里的最优前驱”变成“维护一个窗口里的最优前驱”时，单调队列就是第一候选优化。

## 8. 这组模板最该记住的循环顺序

这是国赛里最容易写挂的地方。

### 8.1 一维线性 DP

常见写法：

- 外层枚举当前位置
- 内层枚举前驱或附加状态

比如 LIS：

- `i` 在外层
- `j < i` 在内层

### 8.2 二维前缀 DP

常见写法：

- `i` 从小到大
- `j` 从小到大

因为当前格只依赖左边、上边、左上角。

### 8.3 背包 DP

一定要区分：

- 0/1 背包：容量倒序
- 完全背包：容量正序
- 多维 0/1 背包：所有费用维都倒序

### 8.4 分组 / 依赖类背包

一定先按“组 / 主件”来推进，再枚举容量。

否则会在同组内部或同主件内部重复选取。

## 9. 常见误区

### 9.1 状态还没定义清楚，就急着写代码

表现：

- 明明是前缀模型，却写成区间模型。
- 明明需要记末尾信息，却只开一维数组。

改法：

- 先用一句中文把 `dp[...]` 的含义说完整。

### 9.2 背包循环顺序写反

这是模板题最常见失误：

- 0/1 背包写成正序，会导致同一件物品被重复选。
- 完全背包写成倒序，会导致当前物品只能选一次。

### 9.3 只会求最值，不会做计数

这组模板里，字符串计数类题提醒你：

- DP 不只是 `max / min`
- 很多时候是“方案数”
- 还有时候是“是否可达”或“最少操作次数”

### 9.4 优化写上了，但没理解为什么能优化

比如：

- LIS 的二分优化，本质是维护每种长度的最优结尾。
- 单调队列优化，本质是窗口最值查询。
- 二进制拆分，本质是把有限重数拆成若干个 0/1 选择。

如果不理解本质，换个题面就不会用了。

## 10. 复习建议

如果按比赛训练顺序复习，这组模板建议这样过：

1. 先重新手写 0/1 背包、完全背包、LIS、编辑距离。
2. 再复习计数类状态设计：不含 101、长度 n 含连续 m 个 1。
3. 再看优化：LIS 贪心二分、单调队列优化、多重背包二进制拆分。
4. 最后复习带结构限制的模型：分组背包、依赖背包、区间覆盖。

最重要的不是把模板背下来，而是每次都问自己三句话：

1. 这个状态到底表示什么。
2. 最后一步从哪里来。
3. 当前循环顺序是否和“每个物品 / 每个位置能否重复使用”一致。

## 11. 一页速记

- LIS：`dp[i]` 常表示“以 i 结尾”。
- 编辑距离：前缀对前缀，最后一步看删插改。
- 计数 DP：状态只记录“影响下一步合法性”的最少信息。
- 0/1 背包：倒序。
- 完全背包：正序。
- 多重背包：二进制拆分后做 0/1。
- 分组背包：按组推进，每组最多一件。
- 依赖背包：先列合法组合，再做背包。
- 单调队列优化：转移只与滑动窗口中的最值有关。
- 滚动数组：当前层只依赖上一层时再压空间。

一句总括：

> DP 的核心不是记住模板，而是先把“状态”和“最后一步”说清楚；一旦这两件事说清楚，代码只是把转移顺序写出来。
